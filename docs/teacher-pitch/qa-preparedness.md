# 教师 Q&A 预案 — 常见问题与标准回答

> 供向老师汇报时参考，预判可能提出的问题并准备标准回答

---

| 问题 | 标准回答 |
|------|---------|
| **为什么不用腾讯云/阿里云？** | 学生项目零预算启动，先使用免费服务验证需求和稳定性。如学校决定推广，需要学校提供服务器资源和备案主体，可立即迁移至腾讯云/学校机房。 |
| **数据是否出境？** | Cloudflare 在中国大陆有 CDN 节点，用户访问数据通过国内节点传输。数据库 D1 的存储位置可配置为境内。如学校有顾虑，可迁移至学校内网服务器，完全关闭公网访问。 |
| **同学隐私怎么保护？** | 系统仅存储基础信息（姓名+宿舍），不涉及身份证号、手机号等强敏感信息。对外接口全部脱敏显示（如"张**"）。操作日志全量审计，任何数据访问留痕。 |
| **谁有权限访问？** | 四级权限体系：学生只读、查寝员可修改状态、教师可审计查看全校数据、管理员可管理全部数据。所有权限变更记录日志。 |
| **如果系统被攻击怎么办？** | 已启用 HTTPS 强制、JWT 鉴权、边缘计算访问控制、安全响应头防护。可进一步接入学校 VPN，关闭公网访问，仅限内网使用。 |
| **聊天记录是否合规？** | 聊天系统已接入敏感词过滤，自动拦截违规内容。聊天仅用于查寝协同，消息不长期存储（房间过期后清理）。 |
| **数据丢了怎么办？** | 支持 Excel 数据导出备份，管理员可定期导出。如迁移至学校服务器，可由信息中心统一做数据库定时备份。 |
| **为什么是一个学生在做？** | 软件开发课程实践 + 创新创业项目，有指导教师把关，代码经过多轮审核和改进。功能已在真实查寝场景中验证。 |
| **系统稳定性如何？** | Cloudflare 全球网络可用性 99.99%，Workers 边缘计算自动扩缩容，无单点故障。如迁移至学校服务器，由信息中心统一保障。 |
| **后续维护谁负责？** | 开发者可提供长期技术支持。如学校推广，建议纳入信息中心日常运维，开发者提供技术文档和交接培训。 |

---

## 附加说明

- 以上回答基于系统当前实际技术实现，无夸大成分。
- 如需补充其他问题或调整措辞，请联系开发者。
- 建议汇报时携带手机/平板现场演示系统功能。
