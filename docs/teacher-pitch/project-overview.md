# 晚寝查寝辅助系统 — 项目情况说明

> 版本：v2026.06 | 1 页纸摘要，供向老师/学校汇报使用

---

## 一、项目背景

本系统由学生自主开发，旨在解决传统纸质查寝方式效率低、易出错、难汇总的问题。查寝员每晚需逐间宿舍核对学生在寝情况，纸质记录后人工汇总上报，流程耗时且数据易丢失。本系统将查寝流程数字化，支持单人/多人协同查寝，实时生成统计报告，大幅提升查寝效率。

**项目性质**：学生软件开发课程实践 + 互联网+创新创业项目，零预算开发，使用 Cloudflare 免费服务验证技术可行性。

---

## 二、技术架构

| 层级   | 技术方案                        | 说明                       |
|--------|--------------------------------|---------------------------|
| 前端   | 静态页面托管于 Cloudflare Pages | 全球 CDN 加速，中国大陆有节点 |
| 后端   | Cloudflare Workers（边缘计算）  | 延迟低，自动扩缩容          |
| 数据库 | Cloudflare D1（SQLite 兼容）    | 数据存储于境内节点          |

**关于 Cloudflare**：Cloudflare（NYSE: NET）为纳斯达克上市公司，在中国大陆部署有 CDN 节点。用户访问时数据通过国内节点传输，不主动出境。

---

## 三、安全举措清单

1. **账号密码 + JWT Token 鉴权**：所有用户需注册登录，Token 30 天有效，防止未授权访问。
2. **四级角色权限**：
   - 学生（只读）：仅可查看查寝结果
   - 查寝员（可修改）：可进行单人/多人查寝操作
   - 教师（可审计）：可查看全校房间状态、历史记录、导出报表
   - 管理员（全权限）：数据管理、系统设置、角色分配
3. **敏感词过滤系统**：聊天内容实时 Trie 树审核，前后端双重拦截，自动禁止违规词汇。
4. **数据脱敏**：对外接口不暴露完整学生姓名（如"张**"），操作日志中姓名掩码处理。
5. **操作日志审计**：所有数据变更、登录、导出操作全量记录，可追溯、不可删除。
6. **教师监管后台**：教师可实时查看全校查寝状态、房间活跃度、历史记录，支持 Excel 报表导出。
7. **HTTPS 强制传输**：全站 HTTPS，启用安全响应头（XSS 防护、点击劫持防护等）。

## 四、数据说明

系统仅存储以下基础信息，属于**弱敏感信息**：

| 字段     | 说明           |
|----------|--------------|
| 学生姓名 | 基础身份标识    |
| 年级     | 如 2023 级     |
| 班级     | 行政班级       |
| 宿舍号   | 如 611         |
| 床号     | 1-6            |
| 查寝状态 | 在寝/离校/事假/外出/未归 |

**不涉及**：身份证号、手机号、家庭住址、成绩、照片等强敏感信息。

---

## 五、迁移承诺

如学校决定正式推广，开发者承诺：

1. **一周内完成迁移**：将系统迁移至学校指定服务器（腾讯云/阿里云/学校机房）。
2. **ICP 备案**：配合学校完成 ICP 备案流程。
3. **学校子域名**：如 `dorm.njpc.edu.cn` 等。
4. **数据迁移**：现有数据完整导出并导入新服务器，零丢失。
5. **技术交接**：提供完整技术文档，协助学校信息中心接管运维。
