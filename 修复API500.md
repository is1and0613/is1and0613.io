# 紧急修复：API 500 [object Response]

## 问题
`_utils.js` 里 `throw errorResponse(...)` 抛出 Response 对象，wrangler 捕获后变成 `[object Response]`，返回 HTML 错误页，前端 JSON 解析失败。

## 修复（两步）

### Step 1：替换 `functions/api/_utils.js` 的 dbGuard 和 verifyToken

**dbGuard 改成抛普通 Error：**
```js
export function dbGuard(env) {
  if (!env || !env.DB) {
    const err = new Error('数据库服务暂未开通，请稍后重试');
    err.status = 503;
    throw err;
  }
}
```

**verifyToken 里所有 `throw errorResponse(...)` 改成抛 Error：**
```js
// 示例：把原来的
// throw errorResponse('未授权', 401);
// 改成
const err = new Error('未授权，请重新登录');
err.status = 401;
throw err;
```

**删掉 verifyToken 里这行：**
```js
if (e instanceof Response) throw e;  // 删除这行
```

**在 `_utils.js` 底部新增包装器：**
```js
export function withErrorGuard(handler) {
  return async function(context) {
    try {
      return await handler(context);
    } catch (e) {
      if (e instanceof Response) return e;
      const status = e.status || 500;
      const message = e.message || '服务器内部错误';
      return errorResponse(message, status);
    }
  };
}
```

### Step 2：给 6 个 API 文件包一层 withErrorGuard

每个文件（auth.js / dorm-data.js / ocr.js / baidu-ocr.js / smart-group.js / deepseek-clean.js）顶部导入：
```js
import { withErrorGuard } from './_utils.js';
```

把导出函数改成：
```js
// 之前
export async function onRequestPost(context) { ... }

// 之后
export const onRequestPost = withErrorGuard(async (context) => { ... });
```

如果文件还有 `onRequestGet` / `onRequestOptions`，同样包：
```js
export const onRequestGet = withErrorGuard(async (context) => { ... });
export const onRequestOptions = withErrorGuard(async (context) => { ... });
```
